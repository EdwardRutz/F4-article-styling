Article #1: What is Git

Git is a version control system (VCS) for tracking changes in computer files and coordinating work on those files among multiple people. It is primarily used for source code management in software development, but it can be used to keep track of changes in any set of files. As a distributed revision control system it is aimed at speed,[9] data integrity, and support for distributed, non-linear workflows

BRANCHES

CREATE BRANCH: git branch SWITCH BRANCHES: git checkout branchname> SEE ALL GIT BRANCHES: git branch -a CREATE A NEW BRANCH AND SWITCH TO IT: git checkout -b DELETE BRANCH: git branch -D HEAD: refers to the latest commit in the current branch git status git add index.html css/ git status (shows files in staging area to be committed) git commit -m "added css" git push origin css-stuff COMMIT ALL + MESSAGE: git commit -a -m “this is my message.” GIT ADD ALL git add -A is equivalent to git add .; git add -u. The important point about git add . is that it looks at the working tree and adds all those paths to the staged changes if they are either changed or are new and not ignored, it does not stage any 'rm' actions. git add -u looks at all the already tracked files and stages the changes to those files if they are different or if they have been removed. It does not add any new files, it only stages changes to already tracked files. git add -A is a handy shortcut for doing both of those.

Article #5: Basic Terminal Commands

Rule #1 of command line

Always know where you are

Basic commands

  • cd - change directory.
  • pwd - print working directory; show where you are.
  • mkdir - make directory.
  • ls - list contents of a directory.
  • clear - clear the view.
  • touch - create file.
  • cat - read the contents of a file.
  • cp - copy files and directories.
  • mv - move files and directories.
  • rm -rf - remove and remove files and folders (delete).
